# 5 分钟录屏视频分镜脚本

## 视频目标
用 5 分钟以内的视频，把项目从真实场景触发，到视觉理解、语音展开、视频陪伴和打卡闭环讲清楚，让评委一眼理解这不是“拍照看报告”，而是“从看见到行动”的完整体验。

## 总体结构
- 0:00 - 0:25 开场与问题
- 0:25 - 1:00 真实场景触发
- 1:00 - 1:55 视觉理解过程
- 1:55 - 2:45 语音追问与行动建议
- 2:45 - 3:45 视频推荐与陪伴式练习
- 3:45 - 4:25 打卡闭环与持续探索
- 4:25 - 5:00 总结与赛道呼应

## 分镜脚本

### 镜头 1：标题页
- 时长：10 秒
- 画面：
  - 项目封面图或海报首页
  - 显示项目名与一句话介绍
- 旁白：
  - 大家好，我们的作品叫做 `中医镜子 · 晨起健康陪伴`。
  - 它是一款围绕“视觉 + 语音”展开的轻健康陪伴体验。

### 镜头 2：问题引入
- 时长：15 秒
- 画面：
  - 切到海报或页面首页
  - 高亮“晨起照镜子”的文案
- 旁白：
  - 很多人早上照镜子时，会感觉自己今天状态不太对，比如脸色差、没精神、上火或疲惫，但通常说不清，也不知道从哪一步开始调节。

### 镜头 3：进入产品首页
- 时长：20 秒
- 画面：
  - 打开项目首页
  - 停留在“视觉触发”“直接说：我今天该怎么调”等核心区域
- 旁白：
  - 所以我们把“照镜子”这个真实动作变成产品入口，让用户先拍一张面部照片，再补一张舌象照片。

### 镜头 4：上传双图
- 时长：25 秒
- 画面：
  - 演示上传面部图
  - 演示上传舌象图
  - 显示状态从未上传变成双图齐全
- 旁白：
  - 这里我们采用双图联合判断。面部图负责气色和神采，舌象图负责舌色、舌苔和津液。系统只分析用户真实上传的维度，不会虚构没有拍到的内容。

### 镜头 5：发起联合初判
- 时长：30 秒
- 画面：
  - 点击“联合初判”快捷按钮或发送问题
  - 展示结果页和结构化结果区
- 旁白：
  - Agent 会先分别解释面部和舌象的线索，再给出联合归纳。这个阶段，AI 的价值不是简单识图，而是帮助用户把模糊感受转成可理解的状态解释。

### 镜头 6：聚焦今日行动建议
- 时长：25 秒
- 画面：
  - 镜头停留在“今日行动建议”
  - 展示“现在先做”“做完再问”“备用路径”
- 旁白：
  - 我们不希望结果停留在静态报告，所以系统会优先告诉用户今天最值得马上做的一件事，并解释为什么推荐这条。

### 镜头 7：语音主路径
- 时长：30 秒
- 画面：
  - 点击“直接说：我今天该怎么调”
  - 或点语音按钮收音
  - 展示问题被填入输入框
- 旁白：
  - 接下来，语音不是附加功能，而是主路径。用户可以直接问“我今天该怎么调”“给我一个现在就能做的动作”，不用大量打字。

### 镜头 8：展示语音回答后的继续建议
- 时长：20 秒
- 画面：
  - 发送语音转写后的问题
  - 展示新一轮回答与建议区
- 旁白：
  - 这样视觉理解就自然展开成语音追问，让 AI 从“告诉你是什么”进一步走向“告诉你接下来做什么”。

### 镜头 9：视频推荐区
- 时长：25 秒
- 画面：
  - 停留在视频推荐卡片
  - 放大“为什么推荐这条”的说明
- 旁白：
  - 结果页下方会继续推荐饮食类和运动类视频，并告诉用户为什么推荐这条内容，这样建议不是孤立的，而是和当前状态直接相关。

### 镜头 10：进入陪伴式练习
- 时长：35 秒
- 画面：
  - 点击“去跟练”
  - 进入播放器弹层
  - 展示准备开始、3 秒倒计时、进度条与反馈
- 旁白：
  - 用户点击后会进入陪伴式练习模式。这里有倒计时、模拟播放、动作跟随进度和鼓励反馈，让用户感觉是在 AI 的陪伴下完成动作，而不是只看了一份报告。

### 镜头 11：完成打卡
- 时长：20 秒
- 画面：
  - 点击“打卡完成”
  - 返回结果页
  - 展示“今日已完成 1 次行动”
- 旁白：
  - 完成后，系统会把“今日已完成 1 次行动”同步回结果页。这一步很关键，因为它说明用户已经从“看见状态”走到了“开始行动”。

### 镜头 12：持续探索
- 时长：20 秒
- 画面：
  - 展示新的建议追问，例如“我刚完成跟练，今晚还要注意什么？”
- 旁白：
  - 打卡后，用户还可以继续追问饮食、晚间安排或第二轮建议，体验不会停在一次性结果，而是持续展开。

### 镜头 13：总结收尾
- 时长：35 秒
- 画面：
  - 回到首页或海报
  - 同时展示流程条或海报总结区域
- 旁白：
  - 所以我们这个作品真正想表达的是：
  - 不是让用户拍照看报告，而是把“照镜子看到自己的状态”变成一次可继续展开的健康对话。
  - 视觉负责看懂我，语音负责和我说清楚，视频和打卡负责陪我做完第一步。
  - 这就是我们对赛道四“从看见到理解与进一步探索”的理解。

## 录屏操作建议
- 录屏前先准备好面部测试图和舌象测试图
- 保证首页打开时代理状态正常
- 演示时优先走主路径，不要在设置页停留
- 如现场网络不稳，可提前切到 Mock 模式录制稳定版
- 打卡环节一定要保留，因为它最能体现完整闭环

## 字幕建议
可在视频中加入 6 句短字幕：
- 晨起照镜子，发现状态不对
- 上传双图，触发视觉理解
- 联合判断，不虚构未拍摄维度
- 直接开口问：我今天该怎么调
- 推荐视频，陪伴式完成第一步
- 打卡后继续探索，而不是停在结果页

## 录制版本建议
- 版本 A：纯录屏 + 旁白
- 版本 B：录屏 + 海报页切场
- 版本 C：录屏 + 结尾镜头展示团队名和项目二维码
