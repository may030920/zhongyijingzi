# 中医镜子 · 晨起健康陪伴

## 一句话介绍
用户在照镜子时拍一张面部照片和一张舌象照片，AI 先完成视觉理解，再通过语音交互给出今日行动建议，并推荐视频跟练与打卡，形成“看见状态 -> 理解状态 -> 开始行动 -> 持续探索”的闭环体验。

## 项目定位
`中医镜子` 是一款围绕“视觉 + 语音”展开的轻健康陪伴作品。它不是传统意义上的静态体质测试工具，而是一个从真实生活触发场景出发的多模态 Agent：用户在晨起照镜子、熬夜后、久坐办公后，感觉自己状态不太对时，可以快速拍下面部和舌象照片，让 AI 先看懂，再继续陪用户完成当天第一步调节行动。

## 目标人群
- 关注身体状态、作息节律和轻养生的普通用户
- 晨起、熬夜后、工位久坐后容易出现疲惫、上火、没精神等状态的人
- 希望获得“今天现在就能做什么”而不是只看抽象报告的人

## 痛点与用户价值
很多人日常都会有一种模糊感受：今天脸色不太对、没精神、口干、上火、疲惫，但往往不知道该怎么描述，也不知道该从哪一步开始调节。

传统健康类产品常见两个问题：
- 要么需要先填写较多问卷，触发成本高
- 要么停留在静态报告，告诉用户“是什么”，却没有把结果自然推进到“接下来做什么”

`中医镜子` 解决的是“从看见到行动”的断层问题：
- 用“照镜子”这个自然动作作为视觉触发入口
- 用面部和舌象联合理解用户当前状态
- 用语音交互降低表达成本，让用户直接问“我今天该怎么调”
- 用行动建议、视频陪伴和打卡闭环，把一次识别结果真正转成即时帮助

## 使用场景
最典型的场景是晨起照镜子。

用户发现自己今天看起来脸色不太对、精神不好，或者感觉有点上火、疲惫，但又说不上具体原因。此时用户上传一张面部照片和一张舌象照片，AI 完成联合判断后，不只给体质倾向，还会直接给出一条最适合当天执行的动作或饮食建议。用户可以进一步通过语音追问，例如：

- “我今天该怎么调？”
- “给我一个现在就能做的动作。”
- “帮我用语音讲一遍今天的饮食建议。”

之后系统会推荐对应视频，用户进入跟练并完成打卡，最后回到结果页继续探索晚间安排、饮食调整或第二轮建议。

## 产品体验流程
1. 用户在真实生活中通过“照镜子”发现自己状态异常
2. 上传面部照片与舌象照片，触发视觉理解
3. AI 分别分析面部气色、神采与舌色、舌苔、津液线索，再做联合归纳
4. 系统优先生成“今日行动建议”，告诉用户先做什么
5. 用户通过语音继续追问具体动作、饮食或作息
6. 系统推荐视频并进入陪伴式跟练流程
7. 用户完成打卡，系统把“今日已完成 1 次行动”同步回结果区
8. 用户继续追问，形成持续探索

## 实现方案
### 视觉理解
- 前端读取面部照片和舌象照片
- 基于本地图像分析提取亮度、红润度、冷暖感、神采、舌色、舌苔、津液等线索
- 双图分析遵循“只分析已上传维度”的原则，不虚构未拍摄内容

### 规则知识库
- 将中医望色、辨神、辨寒热，以及舌色、舌苔、津液规则做成结构化知识库
- 使用规则库辅助解释面部线索、舌象线索和体质映射
- 输出强调“趣味化、规则辅助、非医疗诊断”

### Agent 与语音交互
- 支持本地 Mock、Proxy Agent 与 Remote Agent 三种模式
- 当前默认可接本地代理，连接 OpenAI 兼容后端
- 支持浏览器语音输入与语音播报，让用户不需要大量打字

### 行动闭环
- 根据体质倾向映射本地视频库，推荐饮食类与运动类内容
- 点击视频进入陪伴式播放器
- 播放器包含倒计时、模拟播放、动作跟随进度与鼓励反馈
- 完成后支持打卡，并回写到结果页与后续追问上下文

## 核心创新点
### 1. 从“镜子”出发的真实视觉触发
项目不是为了展示识别能力而设计拍照，而是从“照镜子”这个极自然的日常动作出发，让视觉能力融入真实生活路径。

### 2. 从“识别结果”升级为“行动指南”
我们没有停留在告诉用户“你偏湿热”或“你偏气虚”，而是进一步告诉用户“今天先做什么”，并解释为什么推荐这一条。

### 3. 让语音成为体验主路径
在很多轻健康场景里，用户并不想打很多字。项目把语音前置为主入口，让用户能直接问“我今天该怎么调”，从而自然展开视觉结果后的探索过程。

### 4. 构建完整闭环而非一次性结果
体验不是拍照后结束，而是 `拍照 -> 理解 -> 语音追问 -> 视频跟练 -> 打卡 -> 继续探索`，更贴近赛道四“从看见到理解与进一步探索”的要求。

## 与赛道四的契合点
赛道四强调的是：是否围绕一个真实场景，把从用户触发到体验展开的过程讲清楚，并让人理解技术如何自然发挥作用。

`中医镜子` 的回答是：
- 真实场景：晨起照镜子、熬夜后、工位久坐后
- 视觉能力：从面部和舌象中提取线索，完成状态理解
- 语音能力：让用户不打字也能顺畅追问
- 用户价值：用户立即获得今天可执行的一步建议
- 完整体验：不止知道结果，还能进入视频陪伴、打卡与持续探索

## 关键词说明
### 推荐关键词
`镜子`

### 关键词结合方式
我们的项目正是从“镜子”这个日常动作出发。用户不是为了体验 AI 而刻意拍照，而是在照镜子时自然发现“自己今天状态不太对”，于是拍下面部和舌象照片。镜子既是视觉触发入口，也是整个体验成立的真实场景基础。AI 在镜子场景里先看懂用户状态，再通过语音和视频把结果展开成行动建议，因此“镜子”不是装饰性元素，而是产品体验的起点。
